# 1. 一段话总结
Airweave 是由 airweave.ai 开发的**完全开源的 AI 代理跨应用与数据库上下文检索层**，支持连接 30+ 应用、生产力工具、数据库及文档存储并将内容转化为可搜索知识库，通过 REST API 或 MCP 提供标准化搜索接口；提供**托管服务（Airweave Cloud）和自托管（依赖 Docker 与 Docker Compose）** 两种部署方式，支持 Python、TypeScript/JavaScript  SDK，具备数据同步、实体提取、语义搜索等核心功能，技术栈涵盖 React/TypeScript（前端）、FastAPI（后端）、PostgreSQL 与 Qdrant（数据库）等，当前最新版本为 **v0.7.19（2025年12月4日发布）**，遵循 MIT 许可证。

# 3. 详细总结
## 一、Airweave 核心定位与价值
Airweave 是由 **airweave.ai** 开发的**完全开源的上下文检索层**，专门服务于 AI 代理，核心价值在于打破应用与数据库的数据壁垒：
- 连接范围：支持对接 30+ 类数据源，包括生产力工具、开发工具、业务系统、文档存储等；
- 数据处理：自动将多源数据转化为**可搜索知识库**；
- 接口标准化：通过 **REST API** 或 **MCP**（语义可搜索 MCP 服务器）提供统一访问接口，简化 AI 代理的数据获取流程。


## 二、快速启动指南（两种部署模式）
### 1. 托管服务
直接使用 **Airweave Cloud**，无需本地部署，开箱即用。

### 2. 自托管模式（依赖 Docker 与 Docker Compose）
| 步骤 | 操作命令/说明 |
|------|----------------|
| 1    | 克隆代码仓库：`git clone https://github.com/airweave-ai/airweave.git` |
| 2    | 进入仓库目录：`cd airweave` |
| 3    | 为启动脚本授权：`chmod +x start.sh` |
| 4    | 启动服务：`./start.sh` |
| 5    | 访问仪表盘：http://localhost:8080 |

## 四、使用方式（前端、API、SDK）
### 1. 前端（可视化操作）
- 访问地址：`http://localhost:8080`
- 核心功能：连接数据源、配置数据同步规则、可视化查询数据。

### 2. API（程序调用）
- Swagger 文档地址：`http://localhost:8001/docs`
- 核心功能：创建数据源连接、触发数据同步任务、执行数据搜索。

### 3. SDK（多语言支持）
#### （1）Python SDK
- 安装方式：`pip install airweave-sdk`
- 核心能力（代码示例关键功能）：
  - 创建数据集合；
  - 添加数据源连接（如 Stripe）；
  - 多种搜索模式：语义搜索（默认）、混合搜索（语义+关键词）、带查询扩展与重排序的搜索、带时效性偏向（0.0-1.0，值越高越优先近期数据）的搜索；
  - 结果类型：返回原始结果、返回 AI 生成答案。

#### （2）TypeScript/JavaScript SDK
- 安装方式：`npm install @airweave/sdk` 或 `yarn add @airweave/sdk`
- 核心能力：与 Python SDK 一致，支持集合管理、数据源连接、多模式搜索及多类型结果输出。


## 五、核心功能（6大关键能力）
1. **高效数据同步**：支持 30+ 数据源同步，配置步骤极简；
2. **实体提取与转换**：内置数据处理流水线，自动提取实体、标准化数据格式；
3. **多租户与权限**：采用多租户架构，集成 OAuth2 授权机制，保障数据安全；
4. **增量更新**：基于**内容哈希**实现增量同步，避免重复处理，提升效率；
5. **灵活搜索能力**：支持语义搜索、混合搜索、查询扩展+重排序、时效性偏向搜索4种模式；
6. **数据版本控制**：记录数据变更历史，支持版本回溯。

# 4. 关键问题
## 问题1：Airweave 作为 AI 代理的上下文检索层，其核心优势与同类工具相比体现在哪些方面？
答案：Airweave 的核心优势主要有三点：一是**开源与全功能**，完全开源且支持 30+ 数据源同步，无功能阉割；二是**搜索能力灵活**，除基础语义搜索外，还支持混合搜索、查询扩展+重排序、时效性偏向搜索，且可直接返回 AI 生成答案，满足不同场景需求；三是**部署与集成便捷**，提供托管服务（Airweave Cloud）和自托管（Docker 一键启动）两种模式，同时支持 Python、TypeScript/JavaScript SDK，适配多语言开发环境，降低集成门槛。


## 问题2：在自托管 Airweave 时，若需对接 Stripe 数据源并查询“2024年Q4客户发票”，需通过 Python SDK 执行哪些关键步骤？
答案：需执行4个关键步骤：1. 安装 SDK 并初始化客户端，命令为 `pip install airweave-sdk`，初始化代码需传入 API 密钥与基础地址（如 `client = AirweaveSDK(api_key="YOUR_API_KEY", base_url="http://localhost:8001")`）；2. 创建数据集合，通过 `client.collections.create(name="My Collection")` 生成集合；3. 添加 Stripe 数据源连接，调用 `client.source_connections.create()` 并传入名称（如“My Stripe Connection”）、简称（“stripe”）、集合 ID 及 Stripe API 密钥；4. 执行混合搜索（语义+关键词，适配“2024年Q4”时间关键词），代码为 `results = client.collections.search(readable_id=collection.readable_id, query="customer invoices Q4 2024", search_type="hybrid")`。


## 问题3：Airweave 的技术栈设计如何支撑其“跨应用与数据库的上下文检索”核心需求？
答案：技术栈各组件围绕核心需求形成协同：1. **前端层**（React/TypeScript + ShadCN）保障用户可直观配置数据源与查询，降低操作门槛；2. **后端层**（FastAPI）提供高性能 API 服务，快速处理 AI 代理的检索请求；3. **数据库层**采用“PostgreSQL（元数据）+ Qdrant（向量数据）”组合，PostgreSQL 存储集合、数据源等结构化配置，Qdrant 高效存储向量数据以支撑语义搜索；4. **任务调度层**（Temporal + Redis），Temporal 编排复杂的多源数据同步工作流，Redis 实现实时通信确保增量更新高效；5. **部署层**（Docker Compose/K8s）支持开发与生产环境无缝切换，保障服务在不同规模场景下的稳定性，最终实现跨源数据的高效检索与服务。
